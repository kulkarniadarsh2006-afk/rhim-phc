# RHIM.PHC Portal - Primary Health Centre Simulator

**RHIM.PHC** is a simulation portal designed for Primary Health Centre (PHC) Supervisors, Medical Officers, and Data Entry Operators to manage medicine inventory, track patient OPD count demographics, log seasonal disease trends, and synchronize data with the central **RHIM / MediReach AI** ecosystem.

## Technology Stack & Architecture
- **Backend Framework**: Python Flask
- **Database Layer**: Dual-mode engine:
  - **Local Development**: SQLite (`rhim_phc.db`)
  - **Cloud/Production**: PostgreSQL (activated via `DATABASE_URL` environment variable)
- **UI Styling**: Premium Glassmorphic Vanilla CSS (optimized for responsiveness and modern aesthetics)
- **Visual Analytics**: Chart.js (Monthly Consumption & Seasonal Disease Trends)
- **Icons**: Lucide Icons CDN

## Getting Started (Local Run)

### 1. Prerequisites
Verify that the system has Python. In this environment, the executable is located at:
`C:\Users\DELL\AppData\Local\Python\bin\python.exe`

### 2. Initialize and Seed the Database
Initialize all database tables and seed them with mock health center records (available medicines, test stock items, and historical patient logs):
```bash
C:\Users\DELL\AppData\Local\Python\bin\python.exe database.py
C:\Users\DELL\AppData\Local\Python\bin\python.exe seed.py
```

### 3. Launch the Server
Start the local development server:
```bash
C:\Users\DELL\AppData\Local\Python\bin\python.exe app.py
```
By default, the server spins up at: `http://127.0.0.1:8000/`

## Primary Modules Walkthrough

### 1. Dashboard & Analytics
- Displays key stats (Total Available Medicines, Low Stock, Shortages, Pending Orders).
- Visualizes consumption averages and seasonal disease patterns via interactive line and bar charts.
- Highlights AI recommendations generated by the **RHIM AI Engine** (e.g., predicting seasonal surges and suggesting threshold adjustments).

### 2. Medicine Inventory
- Tracks batch numbers, current stocks, daily consumption, and expiry dates.
- Calculates **Days Remaining = Current Stock / Daily Consumption** on the fly.
- Categorizes stocks via color badges: **Green (Safe)**, **Yellow (Low Stock)**, or **Red (Critical Shortage)**.
- Integrates interactive modals to **Add Batches**, **Edit Details**, **Update Stock Counts** (Add/Dispense/Audit), and **View Transaction Audit Logs** asynchronously.

### 3. Patient OPD & Disease Logging
- Enables daily logging of OPD patient count demographics (Male/Female/Children).
- Records seasonal disease counts (e.g., Malaria, Dengue, Flu) to train the AI's forecasting models.

### 4. Supply Chain Requests
- Permits staff to submit medicine replenishment requests to central warehouses.
- Tracks requests along a five-stage timeline: `Submitted` -> `Under Review` -> `Approved` -> `Dispatched` -> `Delivered`.
- Includes a **Simulate State** panel to advance orders. Transitioning a request to **Delivered** automatically credits the stock count to your local PHC inventory!

### 5. Send to RHIM AI (Synchronization Hub)
- Bundles local clinical data (inventory, OPD loads, and disease spreads) into a unified JSON schema.
- Displays an interactive **Transmission Console** that logs connection steps, TLS checks, payload compilation, and gateway responses.
- Archives every submission in the `ai_submissions` table.

### 6. Reports Hub
- Compiles Daily, Weekly, Monthly, Shortage, and Medicine Consumption reports.
- Renders print-ready preview layouts complete with verification sign-off blocks.
- Provides options to trigger mock spreadsheet downloads or print the generated documents.
